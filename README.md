# 🛴 Sprint_6 — Автотесты для сервиса «Яндекс.Самокат»

Этот проект создан в рамках учебного спринта и покрывает автотестами ключевые сценарии веб-приложения [«Яндекс.Самокат»](https://qa-scooter.praktikum-services.ru) с использованием Selenium, Pytest и Allure.

---

## 📦 Стек технологий

- Python 3.11+
- Selenium WebDriver
- Pytest
- Allure (для отчётов)
- Faker (для генерации фейковых пользователей)

### 🚀 Как запустить тесты
Запуск всех тестов:
```bash
pytest -v
```
Запуск одного файла:
```bash
pytest tests/test_order_flow.py
pytest tests/test_logos.py
pytest tests/test_questions.py
```
С генерацией Allure-отчёта:
```bash
pytest --alluredir=allure-results # 1. Запуск pytest с включением сбора данных для Allure
allure generate allure-results -o allure-report --clean # 2. Генерация HTML-отчета из результатов
allure open allure-report # 3. Открытие сгенерированного отчета в браузере
allure serve allure-results # (Альтернатива пунктам 2 и 3) Одноразовая генерация и отображение отчета
```

### 🔎 FAQ: 

* ❓ Почему ты использовал в одном месте фикстуру дравйвера, а в другом классовые методы setup_class и teardown_class ?
* ❗ Все дело в оптимизации: тесты с вопросами с фикстурой делались более 2 минут. А с классовыми методами 13 секунд.
В рамках 8 вопросов разница не велика, а если открывать придется множество эллементов, счет которых пойдет на десятки уже будет сильней чувствоваться.

---

* ❓ Зачем использовал библиотеку faker? Этого не требовало задание.
* ❗ Я раздумывал о том, какие фикстуры на данный момент были бы актуальны и интересны. И понял, что помимо драйвера можно сделать фейкового пользователя. Это интересно, актуально.

* ❓ Почему в проекте есть закомментированный код? 
* ❗ Это либо настройки, которые можно включить/отключить, либо альтернативные решения задачи. В частности один из тестов  в test_order_flow как раз был сделан по конкретной задаче проекта, а следом за ним более творческое, динамическое решение этой же задачи.
